The ATM Playground supports three installation methods. Choose the one that fits your workflow.

## Prerequisites

Before you begin, ensure your environment meets these requirements:

| Requirement | Minimum version | Notes |
|-------------|-----------------|-------|
| Node.js | 18.x | Required for local and Docker builds |
| npm | 8.x | Bundled with Node.js 18 |
| Browser | Chrome / Edge / Firefox | Webcam access required |
| Webcam | Any | Essential for hand detection |

## Installation methods

<Tabs>
  <Tab title="Local (Node.js)">
    ### Clone and run locally

    **1. Clone the repository**

    ```bash
    git clone https://github.com/googlesamples/mediapipe.git
    cd mediapipe/tutorials/atm_playground
    ```

    **2. Install dependencies**

    ```bash
    npm install
    ```

    **3. Start the development server**

    ```bash
    npm start
    ```

    The app is now available at `http://localhost:3000`.

    **4. Build for production** (optional)

    ```bash
    npm run build
    ```

    This produces an optimized static bundle in `build/` ready for deployment.

    <Note>
      The development server serves the `hand_landmarker.task` model file from `public/` via `http://localhost:3000/hand_landmarker.task`. This local path is the primary source; the CDN is used as a fallback if the local file fails to load.
    </Note>
  </Tab>

  <Tab title="Docker">
    ### Build and run with Docker

    The project includes a multi-stage `Dockerfile` that builds the React app and serves it with Nginx.

    **Dockerfile overview:**

    ```dockerfile
    FROM node:18-alpine as builder
    WORKDIR /app
    COPY . .
    RUN npm ci
    RUN npm run build


    FROM nginx:1.25.1-alpine as production
    ENV NODE_ENV production
    COPY --from=builder /app/build /usr/share/nginx/html
    COPY nginx.conf /etc/nginx/conf.d/default.conf
    EXPOSE 80
    CMD ["nginx", "-g", "daemon off;"]
    ```

    **Build the image**

    From the `tutorials/atm_playground` directory:

    ```bash
    docker build -t atm-playground .
    ```

    **Run the container**

    ```bash
    docker run -p 8080:80 atm-playground
    ```

    Open `http://localhost:8080` in your browser.

    <Warning>
      HTTPS is required for `getUserMedia` (webcam access) in most browsers outside of `localhost`. If you deploy the Docker image behind a reverse proxy, ensure TLS is configured on the proxy layer.
    </Warning>
  </Tab>

  <Tab title="CodeSandbox">
    ### Zero-install browser environment

    No Node.js, no Docker — open the template and start coding immediately.

    **[Open the CodeSandbox template →](https://codesandbox.io/p/sandbox/quizzical-neco-svhglk)**

    The app spins up in the preview panel automatically.

    **To fork and customize:**

    1. Open the template link above.
    2. Press `Ctrl+S` (`Cmd+S` on Mac) — this saves a copy to your CodeSandbox account.
    3. Alternatively, click the **Fork** button in the toolbar.
    4. Edit the source files in the left panel; the preview reloads live.

    <Note>
      Do not modify the `.codesandbox` configuration files — this can break the sandbox environment.
    </Note>

    **Open in a new tab for full webcam access:**

    The embedded preview panel may restrict webcam permissions. Click the "Open in New Tab" button in the preview toolbar for the full experience.
  </Tab>
</Tabs>

<Accordion title="What is @mediapipe/tasks-vision?">
  `@mediapipe/tasks-vision` is the official MediaPipe JavaScript/TypeScript package for vision tasks. It ships the `HandLandmarker`, `GestureRecognizer`, `ObjectDetector`, and other task runners along with their WASM runtime. The ATM Playground uses it exclusively for `HandLandmarker`.
</Accordion>

<Accordion title="Why Workbox?">
  Workbox handles service worker generation and runtime caching strategies. The ATM Playground uses Workbox to precache the `hand_landmarker.task` model file and all static assets, enabling the app to function offline after the first load. See [Offline Support](/guide/offline-support) for details.
</Accordion>
